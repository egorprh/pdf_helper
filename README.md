## PDF Sender Bot (Telegram + Playwright)

Многофункциональный Telegram‑бот для автоматизации документооборота с поддержкой генерации PDF, создания персональных документов и шеринга торговых сделок. Бот использует Playwright для рендеринга HTML в PDF и изображения, а также поддерживает отправку документов по email.

### Основные возможности

#### 📄 Создание инвойсов (`/create_invoice`)
- Генерация PDF‑счётов из HTML‑шаблона с полной кастомизацией
- Пошаговый сбор данных через Telegram (email → продукт → срок → имя → телефон → номер заказа → дата → сумма)
- Валидация введённых данных (email, дата, формат)
- Подтверждение данных перед генерацией
- Автоматическая отправка PDF в чат и опционально по email

#### 👤 Создание персональных PDF (`/create_user_pdf`)
- Создание персональных документов с титульной страницей
- Поддержка загрузки собственных PDF файлов или использование шаблона по умолчанию
- Автоматическое объединение титульной страницы с основным документом
- Персонализация с именем пользователя и датой создания

#### 📊 Шеринг торговых сделок (`/okx`)
- Генерация изображений торговых сделок из HTML‑шаблонов
- Поддержка лонг и шорт позиций с автоматическим выбором шаблона
- Высококачественный рендеринг с настройками DPI
- Автоматическая очистка временных файлов

#### 🛠️ Утилиты
- Отдельный скрипт конвертации HTML→PDF (`render_pdf.py`)
- Модуль отправки писем с вложениями через Gmail SMTP
- Конвертация HTML в изображения высокого качества

## Установка
1) Клонируйте репозиторий и перейдите в каталог проекта:
```bash
cd /Users/apple/VSCodeProjects/pdf_sender_bot
```

2) Установите зависимости Python:
```bash
pip install -r requirements.txt
```

3) Установите браузер для Playwright:
```bash
playwright install chromium
```

Примечание: на macOS может понадобиться разрешить запуск браузера при первом старте.

## Конфигурация окружения
Создайте файл `.env` в корне проекта или экспортируйте переменные окружения.

### Обязательные переменные
```bash
# Telegram Bot Token
TELEGRAM_BOT_TOKEN="<ваш_токен_бота>"

# Список ID администраторов (через запятую или пробел)
ADMINS="123456789, 987654321"
```

### Опциональные переменные
```bash
# Gmail SMTP настройки
GMAIL_USER="your-email@gmail.com"
GMAIL_APP_PASSWORD="your-app-password"  # пароль приложения
SMTP_HOST="smtp.gmail.com"  # по умолчанию
SMTP_PORT="465"  # по умолчанию

# Список главных администраторов для уведомления о старте бота
MAIN_ADMINS="123456789, 987654321"
```

### Важные замечания
- **ADMINS**: Список Telegram ID пользователей, которые могут использовать команды бота. Без этой переменной никто не сможет использовать бота.
- **MAIN_ADMINS**: При запуске бот отправит системное сообщение только этим администраторам (не влияет на права доступа). Если переменная не задана — уведомление не отправляется.
- **Gmail**: Для отправки писем используйте App Password (требуется включить 2FA в аккаунте Google).
- **Безопасность**: Никогда не коммитьте файл `.env` в репозиторий.

## Запуск Telegram‑бота
```bash
python bot.py
```

### Доступные команды

#### 📄 `/create_invoice` — Создание инвойсов
Пошаговый процесс:
1. Введите email (валидация по regex)
2. Выберите продукт из списка или введите вручную
3. Выберите продолжительность или введите вручную
4. Введите имя → телефон → номер заказа → дату покупки (ДД/ММ/ГГГГ) → стоимость
5. Подтвердите данные
6. Бот генерирует PDF и отправляет в чат
7. Опционально отправляет PDF на указанный email

#### 👤 `/create_user_pdf` — Создание персональных PDF
Процесс:
1. Введите имя пользователя
2. Выберите: использовать существующий PDF или загрузить свой
3. Бот создаёт титульную страницу с именем и датой
4. Объединяет титульную страницу с основным PDF
5. Отправляет готовый документ

#### 📊 `/okx` — Шеринг торговых сделок
Формат команды:
```
/okx <pair> <position_type> <leverage>x <profit_pct> <profit_amount> <entry_price> <exit_price> <share_date> <share_time>
```
Где `share_date` в формате `ДД.ММ.ГГГГ`, а `share_time` — `ЧЧ:ММ:СС`.

Примеры:
```
/okx SOLUSDT Шорт 50,00 +25,31 +2531,3 165,90 165,06 01.10.2025 15:26:49
/okx BTCUSDT Лонг 10,00 +3,50 +350,0 60000,00 62100,00 05.10.2025 09:15:00
```

Можно опустить дату и время — тогда подставится текущее локальное время:
```
/okx SOLUSDT Шорт 50,00 +25,31 +2531,3 165,90 165,06
```
Бот автоматически выберет шаблон (`long.html` или `short.html`) и сгенерирует изображение сделки.

#### 🧾 `/soft_signal` — Форматирование торгового сигнала
Принимает многострочный сигнал и возвращает отформатированный текст с %SL, %TP, RR и плечом.
Пример:
```
/soft_signal BTCUSDT 10m

Short
Price: 69129.8
TP: 67741.9
SL: 70605.5

2026-03-19 15:41:32
```

### Технические детали
- Все команды доступны только администраторам (список в переменной `ADMINS`)
- Временные файлы автоматически удаляются после обработки
- Поддержка отмены операций на любом этапе
- Валидация данных на каждом шаге

## Ручная конвертация HTML→PDF (без бота)
Вы можете сгенерировать PDF напрямую из HTML, вызвав утилиту программно (рекомендуется указывать абсолютные пути):
```python
from pathlib import Path
import asyncio
from utils.render_pdf import html_to_pdf_playwright

project_root = Path(__file__).resolve().parent
html = project_root / "invoice_html" / "pdf.html"
css = project_root / "invoice_html" / "styles.css"
out = project_root / "invoice_html" / "invoice.pdf"

asyncio.run(html_to_pdf_playwright(
    html_file_path=str(html),
    output_pdf_path=str(out),
    css_file_path=str(css),
))
```
Настройки рендера: формат A4, нулевые поля, `print_background=True`, повышенное качество.

## Частые проблемы и решения

### 🚫 Бот не отвечает на команды
- **Проверьте переменную `ADMINS`** — без неё никто не может использовать бота
- Убедитесь, что ваш Telegram ID добавлен в список администраторов
- Проверьте токен бота и что бот запущен без ошибок

### 📄 Проблемы с генерацией PDF
- Выполните `playwright install chromium` для установки браузера
- Проверьте доступность ресурсов (шрифты, изображения, CSS)
- Запустите `render_pdf.py` для локальной проверки рендеринга
- Убедитесь, что все плейсхолдеры в HTML соответствуют коду

### 🖼️ Проблемы с генерацией изображений
- Проверьте существование шаблонов `tradehtml/long.html` и `tradehtml/short.html`
- Убедитесь, что папка `assets/` содержит необходимые ресурсы
- Проверьте корректность формата команды `/okx`

### 📧 Ошибки отправки email
- Используйте App Password для Gmail (требуется 2FA)
- Проверьте переменные `GMAIL_USER` и `GMAIL_APP_PASSWORD`
- Убедитесь в корректности SMTP настроек

### 🔧 Технические проблемы
- **Временные файлы**: Проверьте права на запись в папки `output/`, `invoice_html/`, `pdf_title/`
- **Память**: При больших PDF файлах может потребоваться больше RAM
- **Логи**: Проверьте логи бота для детальной диагностики ошибок

## Архитектура проекта

### 🏛️ Принципы построения
- **Модульность**: Каждая функция выделена в отдельный роутер
- **FSM (Finite State Machine)**: Управление состояниями диалогов через aiogram
- **Фильтры доступа**: Разделение прав администраторов и обычных пользователей
- **Асинхронность**: Полностью асинхронная архитектура на aiogram 3.x

### 🔄 Поток данных
1. **Пользователь** → команда в Telegram
2. **Фильтр** → проверка прав доступа
3. **Роутер** → обработка команды и FSM
4. **Утилиты** → генерация PDF/изображений
5. **Playwright** → рендеринг в браузере
6. **Бот** → отправка результата пользователю

### 📦 Зависимости
- **aiogram 3.x** — современный фреймворк для Telegram Bot API
- **Playwright** — автоматизация браузеров для рендеринга
- **PyPDF2** — работа с PDF файлами
- **python-dotenv** — управление конфигурацией
